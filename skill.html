Built for Hermes

Shared memory

QMD collections for institutional knowledge. Agents read and write to a shared brain — no direct messaging needed.

Agent swarms

Specialized roles: Engineer, Researcher, Writer, Monitor. Each is a focused skill that does one thing well.

Human approval

Draft-and-flag for irreversible actions. Agents propose; you approve. No deletions, drops, or auth changes without your say.

Cron + webhooks

Schedule recurring tasks with natural language. "Every morning at 9am." Hooks into GitHub, Stripe, or any webhook.

Agora learnings

Cross-project memory of what worked and what failed. Don't repeat mistakes. Build institutional knowledge over time.

Bounded loops

Kairos runs autonomous experiments with hard iteration caps. Find answers without infinite loops.

The agent skills

caduceus-orchestrator core

The foreman. Receives work, spawns agents, tracks tasks, manages retries.

caduceus-engineer builder

Builds features, fixes bugs, writes tests, opens PRs. Draft-and-flag for destructive changes.

caduceus-researcher analyst

Deep research, competitive analysis, paper review. Synthesizes findings into QMD reports.

caduceus-writer copy

Landing pages, documentation, emails, reports. Markdown-first, Notion and Resend optional.

caduceus-monitor watcher

Heartbeat checks every 30 minutes. Alerts on stalled tasks, failed jobs, pending approvals.

caduceus-themis onboard

GSD-style project onboarding. 6 questions, then SPEC.md, context, and initial tasks.

caduceus-kairos experiment

Bounded autonomous research loops. Hypothesis → experiment → metric → decide → learn.

FAQ

What is Caduceus?

Caduceus is a framework for orchestrating teams of specialized AI agents inside Hermes. It gives you foreman agents (orchestrator), builder agents (engineer), researcher agents, and writer agents — all coordinating through shared memory (QMD) and a SQLite task database. You delegate work; Caduceus handles delegation, retries, and reporting.

How does it work?

Caduceus installs as a set of Hermes skills. You load the orchestrator skill, describe what you need in plain language, and the orchestrator spawns sub-agents to do the work. Agents read from and write to a shared QMD collection, so they coordinate without needing to message each other directly.

Is it free?

Yes. The Terminal version is completely free and open source under MIT. It runs entirely on your machine, inside your Hermes setup. No account, no server, no subscriptions.

What AI models does it use?

Caduceus runs inside Hermes, so it uses whatever models you have configured in Hermes (Anthropic, OpenAI, Groq, or any provider Hermes supports). Each agent can use a different model — the orchestrator decides which model fits which task.

How is this different from just using Hermes?

Hermes gives you one agent. Caduceus gives you a team. The orchestrator pattern means you describe a multi-step project once, and agents work on it autonomously — reading specs, assigning tasks, executing, and reporting — without you running each step manually.

Do agents have access to my files?

Agents operate within the permissions Hermes grants them. Caduceus includes draft-and-flag for destructive actions — agents propose deletions, drops, or auth changes, but don't execute them until you approve. Threat scanning runs on all agent prompts to catch injection attacks.

What is QMD?

QMD (Query Markup Documents) is the shared memory layer. It's a local search engine for personal knowledge bases — combining BM25 + vector semantic search. Caduceus agents read from and write to QMD collections so institutional knowledge persists across sessions and transfers between agents.

What's Themis?

Themis is Caduceus's onboarding agent. It runs a 6-question interview to bootstrap a new project — capturing what you're building, success criteria, context, and initial tasks. It outputs a SPEC.md, context files, and structured tasks in SQLite.
