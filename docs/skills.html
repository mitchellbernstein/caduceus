Skills

Caduceus is a collection of Hermes skills. Each skill is a specialized agent with a defined role, tools, and coordination protocol.

Core skills

caduceus-orchestrator

core spawns other agents

The foreman. Receives work from the user, creates tasks, spawns the right sub-agents, monitors progress via QMD, handles retries and approvals. You talk to the orchestrator — it talks to the rest.

Use it: "run the researcher on competitor analysis"

caduceus-themis

onboard project setup

Runs a GSD-style structured interview to bootstrap a new project. Asks 6 questions, then creates SPEC.md, context.md, and initial tasks in SQLite. One conversation to get a project ready for agents.

Use it: "bootstrap a new project with Themis"

Theoi — agent skills

caduceus-engineer

builder writes code

Builds features, fixes bugs, writes tests, opens PRs. Reads SPEC.md, implements, tests, updates the coordination log. Requires approval for destructive actions (delete, drop, auth changes).

Tools: terminal, read_file, write_file, search_files, git

caduceus-researcher

analyst reads the web

Deep research, competitive analysis, paper review. Searches broadly, synthesizes findings, writes reports to QMD. Multiple sources, dated information, distinguishes fact from opinion.

Tools: web_search, web_extract, arxiv, session_search

caduceus-writer

copy content

Landing page copy, documentation, emails, reports. Markdown-first. Templates for each content type. Optionally sends to Notion or Resend.

Tools: write_file, read_file, web_extract

caduceus-monitor

watcher heartbeat

Periodic health checks every 30 minutes. Alerts on stalled tasks, failed jobs, pending approvals, urgent email. Writes status to QMD state file. Silent if nothing needs attention.

Tools: terminal, session_search, read_file, write_file, cronjob

caduceus-kairos

experiment bounded loops

Runs autonomous experiments with hard iteration caps. Define a hypothesis, run N iterations with metric tracking, decide when to stop (early success or exhaust budget), log learnings to Agora.

Tools: web_search, web_extract, terminal, delegate_task, read_file, write_file

How agents coordinate

Agents don't message each other directly. They write to QMD:

## 2026-03-19

- [engineer] Completed: build landing page → Output: projects/ugc/landing.md
- [researcher] Started: competitor analysis
- [writer] Blocked: waiting on engineer (needs structure)

Other agents read the coordination log to see what's happening. No message bus, no direct communication — just a shared brain in QMD.
